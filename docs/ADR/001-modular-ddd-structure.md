# ADR-001: 모듈형 도메인 주도 구조 채택

## 상태
승인됨 (Accepted)

## 배경
어드민 콘솔의 확장성과 유지보수성을 보장할 수 있는 폴더 구조가 필요합니다. 표준 Next.js의 평면적인 구조는 기능이 늘어남에 따라 코드가 복잡해지는 경향이 있습니다.

## 결정
`src/` 내에 **도메인 주도 설계 (DDD)** 기반의 모듈형 구조를 채택합니다:
- `features/`: 고수준 비즈니스 모듈.
- `entities/`: 도메인 모델 및 비즈니스 로직.
- `shared/`: 재사용 가능한 제네릭 프리미티브.
- `widgets/`: 복잡한 UI 블록.

## 결과
- **장점**: 코드 가독성 및 탐색성 향상, 테스트 용이성, 관심사 분리 명확화.
- **단점**: 초기 설정 비용 및 신규 개발자의 학습 곡선 발생 가능성.
