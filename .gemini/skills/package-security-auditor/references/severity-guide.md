# 패키지 보안 취약성 정의 (Severity Guide)

본 가이드는 `npm audit`에서 분류하는 취약점의 심각도를 정의하고 대응 방안을 제안합니다.

## 🔴 Critical (심각)
- **정의**: 공격자가 원격에서 코드를 실행하거나 시스템을 완전히 장악할 수 있는 취약점입니다.
- **대응**: 즉시 업데이트가 필요합니다. 하위 호환성이 깨지더라도 최신 버전으로의 마이그레이션을 강력히 권장합니다.

## 🟠 High (높음)
- **정의**: 서비스 거부(DoS), 민감한 정보 유출, 또는 특정 조건하의 원격 코드 실행이 가능한 취약점입니다.
- **대응**: 가능한 한 빨리 업데이트해야 합니다.

## 🟡 Moderate (중간)
- **정의**: 복잡한 조건이 충족되어야 하거나 공격 영향력이 제한적인 취약점입니다.
- **대응**: 정기적인 유지보수 일정 내에 업데이트를 계획하십시오.

## 🔵 Low (낮음)
- **정의**: 공격 성공률이 매우 낮거나 피해가 미미한 취약점입니다.
- **대응**: 다음 마이너/패치 업데이트 시 함께 진행하십시오.
