# 패키지 보안 취약성 검사 (Package Security Auditor)

이 스킬은 프로젝트의 의존성 패키지에서 발생한 보안 취약점을 식별하고 대응 방안을 제시하는 워크플로우를 제공합니다.

## 사용 시점
- "현재 프로젝트의 패키지 보안 취약성을 점검해줘."
- "보안 리포트를 생성해줘."
- "어떤 패키지에 업데이트가 필요한지 확인해봐."

## 워크플로우

1.  **환경 확인**: `package-lock.json` 파일이 존재하는지 확인합니다.
2.  **검사 실행**: `npm audit` 명령어를 실행하여 취약점 데이터를 수집합니다.
3.  **보고서 생성**: 수집된 데이터를 바탕으로 다음 형식의 **한글 리포트**를 작성합니다.
    - 취약점 요약 (심각도별 개수)
    - 주요 위험 패키지 목록 및 취약점 설명
    - 권장 조치 사항 (예: `npm audit fix`)

## 참고 가이드
상세한 심각도 정의 및 대응 가이드는 다음 파일을 참조하십시오:
- [보안 취약성 정의 (Severity Guide)](references/severity-guide.md)

## 실행 예시 (Pseudocode)
```bash
# 취약점 검사 및 요약 보고서 생성
npm audit
```
*참고: 상세한 JSON 분석이 필요한 경우 `npm audit --json`을 사용하십시오.*
